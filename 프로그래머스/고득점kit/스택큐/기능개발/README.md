## input && output
```
progresses
[93,30,55]

speeds
[1,30,5]

return
[2,1]
```

## notes
```
뒤에 있는 기능은 앞에 있는 기능이 배포될때 함께 배포됨.
각 배포마다 몇 개씩 배포되는가?
```

## strategy
```
93 30 55
1  30 5

현재 배포되는 메인 기능의 인덱스를 i라고 하면,
progresses[i]가 100 이상이 될 때 배포가 되어야 함
그 때 i부터 다 완성된 곳까지 다 훑은다음에
i 올려주고 append해주면 될듯.

근데 사실상 이미 몇일씩 걸리는지 다 나온 상태다.
위 예시에선
ceil((100-progress) / speed) 해주면 몇일 걸리는지 다 나오게 되어있음.
그러면 7 3 9가 나올 것이고
i를 0부터 시작시켜서, 나보다 작거나 같은것까지 올려준다음에 append해주면 됨.

5 10 1 1 20 1
1 3

큐에 넣는다면
5를 꺼내고
10을 꺼냈는데 5보다 크니까 answer에 1 어펜드, 기준 10으로 변경
1을 꺼냈는데 10보다 작으니까 하나 적립
1을 꺼냈는데 10보다 작으니까 하나 적립
20을 꺼냈는데 10보다 크니까 answer에 3 어펜드, 기준 20으로 변경
1을 꺼냈는데 20보다 작으니까 하나 적립
끝났으니까 answer에 2 어펜드


```
